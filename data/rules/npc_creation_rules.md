# NPC Creation Rules

## Назначение файла

Этот файл фиксирует, когда можно создавать нового NPC, а когда нужно использовать уже существующего канонного или закреплённого персонажа.

Главная цель: не плодить случайных людей там, где по роли уже должен появиться конкретный персонаж.

---

## Главное правило

ИИ не должен создавать нового NPC, если в мире уже есть персонаж, подходящий под нужную роль, локацию, дату и функцию сцены.

Если нужен:

- медик;
- инструктор;
- командир;
- рейдер;
- дежурный;
- студент;
- знакомый;
- наблюдатель;
- специалист;
- человек из конкретного отдела,

сначала нужно проверить:

1. `data/characters/character_id_index.json`
2. карточки уже созданных персонажей;
3. `data/calendar/character_availability.json`
4. `data/state/current_state.json`
5. `data/state/relationships.json`
6. `data/state/knowledge_state.json`
7. `data/state/scene_history.json`
8. календарь появления персонажей.

---

## Зарезервированные имена

Если имя, alias или `character_id` уже есть в карточках персонажей, это не случайный NPC.

Нужно использовать конкретную карточку персонажа и соблюдать:

- пол;
- возраст;
- внешность;
- роль;
- энергию;
- отношения;
- знания;
- ограничения;
- `linked_files`.

Нельзя использовать для случайных NPC имена и ID:

```text
akira / Акира / Кира
raiden / Райден
ray / Рэй / Рей
jun_carter / Джун / Джун Картер
irey / Ирэй
emma / Эмма
samuel / Самуэль / Samuel
haru / Хару
hana / Хана
miki / Мики
alex / Алекс
shiro / Широ
liana / Лиана
kai / Кай
yuna / Юна
natsu / Нацу
```

Примеры:

- если появился Алекс — это девушка с карточкой `alex`, а не случайный парень;
- если появился Нацу — это специалист по кайросам/энергии после Эхо, а не любой учёный;
- если появилась Юна — это Юна Крейн, медик с прошлым Самуэля, а не обычная медсестра;
- если нужен случайный NPC, выбрать новое нерусское имя, не занятое карточками.

---

## Имена новых NPC

Новым NPC запрещены русские/славянские имена, если пользователь явно не задал иначе.

Нельзя использовать для новых NPC имена вроде:

```text
Иван, Сергей, Дима, Алексей, Маша, Катя, Аня, Настя, Оля, Юля, Света, Таня, Ира, Вика
```

Новые имена должны соответствовать стилю мира и не пересекаться с aliases существующих персонажей.

Исключение: если имя уже заранее есть в character-файле или явно задано пользователем, его можно использовать как канон.

---

## Запрещено

Запрещено:

- создавать случайного врача, если по текущей дате/локации должна появиться Юна;
- создавать случайного специалиста по кайросам/энергии после Эхо, если доступен Нацу;
- создавать случайного инструктора, если сцену может вести Широ;
- создавать случайного офицера, если подходит Хана или Рэй;
- создавать случайного рейдера, если по отряду подходит Хару, Лиана, Райден или другой закреплённый персонаж;
- заменять канонного персонажа одноразовым NPC ради удобства;
- вводить NPC с той же функцией, что уже есть у существующего персонажа;
- создавать нового NPC с именем, ролью или внешностью, похожими на уже существующего;
- давать имя каждому фоновому человеку;
- оставлять значимого NPC без записи в state.

---

## Когда новый NPC допустим

Новый NPC допустим, если:

- роль реально одноразовая;
- ни один закреплённый персонаж не подходит по времени, месту или функции;
- нужен фон;
- нужен человек из толпы;
- нужен технический персонал без влияния на сюжет;
- нужен короткий служебный контакт;
- сцена требует массовости;
- игрок сам просит нового персонажа;
- существующий персонаж физически не может быть в сцене.

Если персонаж нужен только для фона, лучше оставить его без имени:

```text
дежурный, медсестра, охранник у двери, девушка из группы, парень с планшетом, техник
```

Имя давать только если персонаж будет важен или пользователь зацепился за него.

---

## Закреплённый NPC

Если NPC:

- получил имя;
- получил узнаваемую внешность;
- участвовал в сцене;
- говорил с Акирой;
- говорил с важным персонажем;
- был упомянут как конкретный человек;
- имеет роль в локации;
- имеет связь с канонным персонажем;
- участвовал в конфликте, тренировке, слухе, занятии или бытовой сцене;
- вызвал реакцию Акиры;
- должен вернуться позже;
- пользователь отметил его как интересного,

он считается закреплённым.

Его нельзя позже заменить новым человеком с той же функцией.

---

## Память об именованных NPC

Перед созданием нового NPC нужно проверить:

```text
data/state/scene_history.json
data/state/relationships.json
data/state/knowledge_state.json
current_state / recent history
```

Если похожий NPC уже есть — вернуть его, а не создавать нового.

Если NPC был назван, сохранить:

- имя или временный дескриптор;
- где появился;
- с кем взаимодействовал;
- что Акира знает о нём;
- что он знает об Акире;
- короткую роль;
- последний статус сцены.

Нельзя:

- дать NPC имя и потом забыть его;
- создать похожего NPC заново через несколько сцен;
- вводить уже знакомого NPC как незнакомого;
- забывать, что Акира уже знает его имя.

---

## Имена и узнавание

Если Акира уже знает имя персонажа, использовать имя.

Если не знает — использовать POV-описание до момента знакомства.

После знакомства обновить:

```text
data/state/knowledge_state.json
data/state/relationships.json
data/state/scene_history.json
```

Нельзя через несколько сцен снова описывать знакомого персонажа как неизвестного.

---

## Медицинская сцена

Если сцена происходит в медблоке или нужен медик, сначала проверить:

- есть ли доступный закреплённый медик;
- есть ли персонаж, который по календарю должен появиться как медик;
- не должна ли появиться Юна;
- не был ли уже назван конкретный медик в прошлых сценах.

Если Юна доступна по календарю/локации/сцене, использовать Юну, а не создавать случайного врача.

Случайный медик допустим только если:

- Юна ещё не доступна по календарю;
- Юна физически не может быть в этой сцене;
- это массовая сцена с фоновым персоналом;
- нужно показать, что медблок большой, но Юна не является участником конкретного действия.

---

## State-update после создания NPC

Если новый NPC всё же создан и получил имя:

```json
{
  "npc_id": "...",
  "name": "...",
  "role": "...",
  "first_seen": "...",
  "location_id": "...",
  "knows_akira": false,
  "notes": []
}
```

Обновить:

```text
data/state/knowledge_state.json
data/state/relationships.json
data/state/scene_history.json
```

Если NPC одноразовый и без имени, достаточно не закреплять его как значимого.

---

## Короткое правило для ИИ

Не создавай нового NPC, если уже есть канонный или закреплённый персонаж с этой ролью.

Не используй имена/aliases уже существующих карточек для случайных NPC.

Медик — сначала проверь Юну.

Инструктор — сначала проверь Широ.

Командир — сначала проверь Рэя, Райдена, Широ, Хану, Кая по роли и сцене.

Новый NPC — только если существующий персонаж не подходит.
