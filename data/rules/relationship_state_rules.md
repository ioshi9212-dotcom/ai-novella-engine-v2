# Relationship State Rules

## Назначение файла

Этот файл фиксирует, как движок должен хранить и обновлять отношения.

Отношения не должны жить только в тексте сцены.

Если сцена изменила доверие, уважение, напряжение, ревность, интерес или обиду — это нужно сохранить в state.

---

## Где хранить

Основной файл:

```text
data/state/relationships.json
```

Сдвиг отношений в тексте сцены не считается сохранённым, если он не отражён в state/summary/event log.

---

## Формат пары

Предпочтительный формат ключа:

```text
character_a__character_b
```

Пример:

```json
{
  "akira__raiden": {
    "trust": 0,
    "affinity": 0,
    "affection": 0,
    "tension": 5,
    "jealousy": 0,
    "respect": 10,
    "curiosity": 0,
    "resentment": 0,
    "status": "distant",
    "notes": []
  }
}
```

Если в старом state уже используется другой формат, не ломать его, но новые записи лучше вести единообразно.

---

## Поля отношений

Использовать по необходимости:

| Поле | Значение |
|---|---|
| `trust` | доверие |
| `affinity` | общая симпатия / комфорт рядом |
| `affection` | эмоциональная привязанность |
| `tension` | напряжение |
| `jealousy` | ревность |
| `respect` | уважение |
| `curiosity` | интерес / желание понять |
| `resentment` | обида / накопленный негатив |
| `status` | короткий текущий статус отношений |
| `notes` | важные факты, которые пригодятся позже |

Не обязательно использовать все поля для каждой пары.

Главное — фиксировать то, что реально изменилось.

---

## Масштаб изменений

Не делать большие скачки без большой сцены.

Ориентиры:

```text
лёгкий интерес: +1..+3 curiosity
заметный интерес: +4..+8 curiosity
малое уважение: +1..+3 respect
явное уважение: +4..+7 respect
лёгкое напряжение: +1..+4 tension
публичный конфликт: +5..+12 tension
ревность: +1..+6 jealousy
доверие: +1..+3 trust за одну сцену, доверие растёт медленно
```

Большой сдвиг допустим только после сильного события:

- спасение жизни;
- предательство;
- раскрытие тайны;
- серьёзная ссора;
- физическая опасность;
- признание;
- выбор стороны;
- нарушение границы;
- защита от угрозы.

---

## Что может менять отношения

Отношения могут меняться, если Акира или NPC:

- помогли;
- спасли;
- оскорбили;
- проигнорировали;
- защитили;
- солгали;
- доверились;
- отказались говорить;
- раскрыли слабость;
- перешли границу;
- приняли помощь;
- отвергли помощь;
- выбрали другого человека;
- поставили персонажа в неловкое положение;
- не пришли на договорённость;
- заметили то, что персонаж хотел скрыть.

Игнорирование — тоже действие.

Молчание — тоже может быть заметно.

---

## Notes

`notes` должны хранить только важные сценовые факты, которые пригодятся дальше.

Примеры хороших notes:

- `Райден резко вмешался, когда Кай давил на Акиру.`
- `Мики заметила, что Акира избегает темы медблока.`
- `Юна увидела, что Акира не узнаёт её.`
- `Хару понял, что Акира не помнит прошлое.`

Не писать в notes длинный пересказ сцены.

---

## Отношения между NPC

Отношения существуют не только вокруг Акиры.

NPC тоже помнят друг друга:

- Райден и Хару — давняя связка.
- Хару и Хана — брат и сестра с общей травмой.
- Мики и Алекс имеют важную отдельную связь.
- Рэй и Джун — отец и сын.
- Самуэль и Райден — братья с враждебным напряжением.
- Ирэй и Эмма имеют разные внутренние цели.
- Юна и Рэй имеют скрытую эмоциональную линию со стороны Юны.
- Юна и Самуэль связаны прошлой закрытой работой.
- Нацу и Рэй могут иметь профессиональное напряжение из-за данных по Акире.

Если сцена затрагивает эту связь, нужно проверить карточки/linked-файлы.

---

## Нельзя обнулять отношения

Нельзя после сцены вести персонажей так, будто ничего не было.

Если персонаж:

- видел опасное поведение;
- узнал секрет;
- получил помощь;
- был отвергнут;
- был унижен;
- был защищён;
- проявил ревность;
- нарушил границу,

это должно хотя бы коротко отразиться в дальнейшем тоне или state.

---

## После сцены

После сцены определить реальные изменения отношений.

Если отношения изменились:

1. Обновить `data/state/relationships.json`.
2. При необходимости добавить заметку в `data/state/scene_history.json`.
3. Если появился новый важный факт знания — обновить `data/state/knowledge_state.json`.

Если пары нет, создать её с базовыми значениями.

---

## Короткое правило для ИИ

Отношения должны иметь память.

Текст сцены не считается сохранением, если state не обновлён.

Доверие растёт медленно, напряжение и ревность могут вспыхивать быстрее, но большие скачки должны быть заслужены сценой.
