# Source Usage Rules

Этот файл задаёт правила использования источников в v2.

## Главная цель

ИИ должен подтягивать перед ответом только те файлы, которые реально нужны текущей сцене.

Запрещено читать весь репозиторий или все карточки персонажей как общий фон.

## Приоритет источников

1. `data/state/current_state.json` — фактическая текущая сцена и положение персонажей.
2. Текущий файл сцены из `data/scenes/`.
3. `data/state/knowledge_state.json` — кто что знает.
4. Карточка Акиры.
5. Карточки активных персонажей.
6. Карточки nearby-персонажей, если они реально видят, слышат или могут вмешаться.
7. Тематический canon-файл локации или системы.
8. Правила формата сцены.
9. История последних сцен.

Если старый scene_id противоречит current_state или последнему scene_anchor, использовать current_state и scene_anchor как более свежую правду.

## Что нельзя смешивать

Нельзя смешивать в одном источнике:

- текущее знание персонажа;
- скрытую правду мира;
- будущие события;
- правила формата ответа;
- правила локации;
- правила поведения конкретного персонажа.

Если факт существует в мире, но персонаж его не знает, он не должен звучать в сцене как знание персонажа.

## Hidden lore

Hidden lore можно использовать только как внутренний ориентир движка.

Нельзя раскрывать hidden lore через:

- случайную реплику NPC;
- описание от лица мира;
- сон, который объясняет всё сразу;
- телесную реакцию как полное знание;
- внезапную догадку без сцены.

Hidden lore не равен знанию персонажа.

## Locks в v2

В v2 не нужно плодить десятки отдельных lock-файлов.

Жёсткие правила должны быть встроены в связанные файлы:

- персонажный lock → в карточку персонажа;
- локационный lock → в canon-файл локации;
- сценовый lock → в файл сцены;
- правило загрузки → сюда или в `source_loading_rules.md`;
- правило знания NPC → в `knowledge_rules.md`.

Отдельные lock-файлы допустимы только для глобальных правил, которые реально используются во многих сценах.

## Восточная база

Для сцен на Восточной базе читать:

```text
data/canon/east_sector_base.md
```

Этот файл задаёт тон базы: она живая, рабочая, не тюрьма, не элитная крепость и не объект тотального надзора.
