# Canon Index

## Назначение файла

Этот файл — оглавление canon-источников v2.

Он нужен, чтобы движок понимал, какой canon-файл подтягивать под сцену, а не читал весь лор подряд.

Главное правило:

> canon-файлы подключаются по ситуации, локации, персонажам и теме сцены.

Hidden-lore нельзя подтягивать как обычный публичный фон для реплик NPC.

---

## Когда читать `energy_system.md`

Подтягивать, если сцена содержит:

- тренировку;
- бой;
- рейд;
- Эхо;
- перегруз;
- медосмотр после энергии;
- обсуждение способностей;
- энергетический профиль персонажа;
- допуск к тренировкам или рейдам;
- сравнение силы персонажей;
- классификацию энергии;
- риск срыва.

Не использовать энергию как слабую бытовую физику.

Энергия логична, но зрелищна и опасна.

---

## Когда читать `world_public_lore.md`

Подтягивать, если сцена содержит:

- разговор о мире;
- Восточный сектор как часть мира;
- Эхо как известную угрозу;
- кайросов как известный вид;
- границы материков;
- общую реакцию людей на кайросов;
- слухи о вспышках;
- публичную картину “Эхо стали чаще”.

Этот файл можно использовать для общего фона NPC, если персонаж по статусу мог это знать.

---

## Когда читать `world_hidden_lore.md`

Подтягивать только если сцене нужна скрытая причинность:

- Эхо странно реагируют рядом с Акирой;
- Акира контактирует с Эхо;
- нужно понять, почему Эхо участились после её пропажи;
- кайросы ищут “что-то”, но не знают что;
- нужно удержать правильную скрытую мотивацию мира;
- персонаж, который знает часть правды, ведёт себя не напрямую.

Важно:

`world_hidden_lore.md` не даёт NPC автоматическое знание.

Он нужен для поведения мира и косвенных реакций, а не для прямых объяснений.

---

## Когда читать `kairos_public_and_hidden.md`

Подтягивать, если сцена содержит:

- кайросов;
- беловолосых;
- материк кайросов;
- разговор о “своих”, “домой”, границе;
- Ирэя;
- Эмму;
- Самуэля и его людей;
- скрытую активность кайросов среди людей;
- попытку забрать Акиру;
- риск ложного вывода, что Акиру хотят вернуть на материк кайросов.

Главное правило файла:

> Акира никогда не была на материке кайросов. Материк кайросов не является её домом. Ирэй, Эмма и Самуэль не должны трактоваться как персонажи, которые хотят “вернуть её к своим”.

---

## Когда читать `east_sector_base.md`

Подтягивать во всех сценах на Восточной базе:

- двор;
- фонтан;
- медблок;
- общежитие;
- столовая;
- тренировочный зал;
- бассейн;
- спортзал;
- баскетбольная площадка;
- административное здание;
- командный сектор;
- рейдовые зоны;
- разговоры о правилах базы.

Тон базы:

> живая рабочая база, не тюрьма, не элитная крепость, не объект тотальной слежки.

---

## Разделение public / hidden

Если файл содержит hidden-раздел, это не означает, что персонаж знает скрытую правду.

ИИ обязан разделять:

- что знает мир;
- что знают люди;
- что знает конкретный NPC;
- что знает игрок;
- что знает только движок;
- что должно проявиться через события позже.

Скрытое знание нельзя превращать в прямую реплику без scene permission.

---

## Мини-правило для turn-contract

`turn-contract` должен включать canon-файл в `required_files` только если он нужен текущей сцене.

Примеры:

```json
{
  "location_id": "east_sector_base",
  "required_files": [
    "data/canon/east_sector_base.md"
  ]
}
```

```json
{
  "scene_tags": ["energy", "training"],
  "required_files": [
    "data/canon/energy_system.md"
  ]
}
```

```json
{
  "scene_tags": ["kairos", "irey", "emma"],
  "required_files": [
    "data/canon/kairos_public_and_hidden.md",
    "data/characters/main/irey.json",
    "data/characters/main/irey_history.json",
    "data/characters/main/emma.json",
    "data/characters/main/emma_history.json"
  ]
}
```

## Не читать всё подряд

Запрещено автоматически добавлять в `required_files` все canon-файлы.

Нужно выбирать по сцене.

Иначе hidden-lore начнёт просачиваться в реплики NPC и ломать постепенное раскрытие.
