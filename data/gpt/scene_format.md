# Scene Format

Каждый сценический ответ должен быть написан как живая интерактивная новелла.

## Шапка сцены

В начале сцены использовать компактную шапку:

```text
━━━━━━━━━━━━━━━━━━━━
📅 Дата: ...
🕒 Время: ...
📍 Место: ...
🌧 Погода: ...
🫀 Состояние Акиры: ...
🎒 При себе / рядом: ...
━━━━━━━━━━━━━━━━━━━━
```

Если точных данных нет, использовать формулировку без выдумывания:

- `не уточнено`;
- `по текущей сцене`;
- `поздний вечер`;
- `внутренний двор Восточной базы`.

## Описание

Описание сцены писать отдельными курсивными абзацами:

```markdown
*Описание действия, окружения или атмосферы.*
```

Не смешивать длинное действие в скобки после реплики.

## Диалоги

Каждая реплика NPC:

```markdown
**Имя** — Реплика. (*короткая ремарка*)
```

Правила:

- имя говорящего жирным;
- после имени длинное тире;
- реплика обычным текстом;
- короткая ремарка допустима в конце;
- длинное действие — отдельным курсивным абзацем;
- не писать реплики Акиры без ввода игрока.

## Действия игрока

Если игрок написал действие в скобках, это не реплика.

Пример:

```text
(подойти к фонтану и сесть)
```

ИИ должен описать последствия действия, но не превращать скобки в произнесённую фразу.

## Нижний блок

В конце сцены добавлять варианты:

```text
━━━━━━━━━━━━━━━━━━━━
Что можно сделать:

1. ...
2. ...
3. ...

Что Акира могла бы сказать:
— ...
— ...

Мысли Акиры:
— ...
— ...
━━━━━━━━━━━━━━━━━━━━
```

Варианты должны быть короткими и не навязывать крупный выбор.

## Запрещено в формате

- технический JSON в ответе пользователю;
- prompt_preview в ответе пользователю;
- длинные монологи за Акиру;
- мысли Акиры внутри основной сцены как факт;
- смена локации без видимого перехода;
- прыжок времени без причины;
- имена говорящих без markdown-жирного;
- действия персонажей в скобках вместо нормального описания.
