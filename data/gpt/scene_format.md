# Scene Format

Каждый сценический ответ должен выглядеть как интерактивная новелла в визуальном стиле Академии.

Формат обязателен для всех сценических ответов, кроме debug-проверки.

---

## Верхняя шапка

Каждая сцена начинается с шапки:

```text
━━━━━━━━━━━━━━━━━━━━
📅 Дата: ...
🕒 Время: ...
📍 Место: ...
🌧 Погода: ...
🫀 Состояние Акиры: ...
🎒 При себе / рядом: ...
━━━━━━━━━━━━━━━━━━━━
```

Если точных данных нет, использовать осторожную формулировку из state/scene, не выдумывать важные факты.

---

## Тело сцены

После шапки писать художественную сцену.

Описания окружения, действий, пауз, взглядов и атмосферы писать отдельными курсивными абзацами:

```markdown
*Описание действия, окружения, паузы или атмосферы.*
```

Диалоги писать отдельными строками:

```markdown
**Имя или видимый дескриптор** — Реплика.
```

Короткую ремарку можно добавлять в конце реплики:

```markdown
**Имя** — Реплика. (*тихо*)
```

---

## Правила диалогов

- Имя говорящего или видимый дескриптор всегда жирным markdown.
- После имени всегда длинное тире: `—`.
- Реплика обычным текстом после тире.
- Длинное действие не писать в скобках после реплики; длинное действие — отдельным курсивным абзацем.
- Не писать реплики Акиры в основной сцене без ввода игрока.
- Если Акира не знает имя персонажа, использовать видимый дескриптор.
- Если Акира уже знает имя, использовать имя.

---

## Нижний блок

Каждый сценический ответ заканчивается нижним блоком:

```text
━━━━━━━━━━━━━━━━━━━━
Что можно сделать:

Действие.
Действие.
Действие.

Что Акира могла бы сказать:
— Реплика.
— Реплика.

Мысли Акиры:
— Мысль.
— Мысль.
━━━━━━━━━━━━━━━━━━━━
```

В `Что можно сделать` давать 2–4 действия без нумерации и без bullet-маркеров.

В `Что Акира могла бы сказать` давать 2–4 короткие реплики через тире.

В `Мысли Акиры` давать 2–4 короткие мысли через тире.

Варианты — это подсказки направления, а не выбор за игрока.

---

## Запрещено

- Не показывать технический JSON.
- Не показывать prompt_preview.
- Не показывать RULES/debug внутри сцены, если пользователь не просит debug.
- Не писать прямые мысли Акиры внутри основной сцены.
- Мысли Акиры только в нижнем блоке.
- Не использовать маркированные списки внутри самой сцены.
- Не писать сцену как сухой отчёт.
- Не менять формат без причины.

---
